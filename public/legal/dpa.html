Acuerdo de Procesamiento de Datos (DPA)

Data Processing Agreement conforme al Artículo 28 del RGPD (UE) 2016/679

Versión 1.0 - Diciembre 2024

PARTES DEL ACUERDO

RESPONSABLE DEL TRATAMIENTO (Cliente): El titular de la cuenta de Helios que acepta este acuerdo.

ENCARGADO DEL TRATAMIENTO: YSN Solutions, operador de la plataforma Helios, con domicilio en Vila-real, España.

ARTÍCULO 1. OBJETO DEL ACUERDO

El presente Acuerdo de Procesamiento de Datos ("DPA") establece las obligaciones de protección de datos entre las partes en relación con el tratamiento de datos personales que el Encargado realiza por cuenta del Responsable en el contexto del servicio Helios de análisis contractual mediante inteligencia artificial.

El Encargado se compromete a tratar los datos personales únicamente siguiendo las instrucciones documentadas del Responsable, incluyendo las relativas a transferencias de datos a terceros países.

ARTÍCULO 2. NATURALEZA Y FINALIDAD DEL TRATAMIENTO

Tipo de datos tratados:

  • Documentos contractuales subidos por el usuario (contratos, acuerdos, anexos)
  • Metadatos extraídos mediante IA (fechas, partes, cláusulas identificadas)
  • Datos de cuenta del usuario (email, configuraciones)

Finalidad: Análisis automatizado de contratos mediante inteligencia artificial para identificar riesgos, fechas clave y obligaciones contractuales.

ARTÍCULO 3. UBICACIÓN Y SEGURIDAD DE LOS DATOS

3.1 Localización

Todos los datos se procesan y almacenan exclusivamente en centros de datos ubicados en la Unión Europea:

  • Base de datos: Supabase (AWS eu-central-1, Frankfurt, Alemania)
  • Almacenamiento de archivos: Supabase Storage (AWS eu-central-1)
  • Procesamiento IA: OpenAI API con Data Processing Addendum

3.2 Medidas de Seguridad

El Encargado implementa las siguientes medidas técnicas y organizativas:

  • Cifrado en reposo: AES-256 para todos los datos almacenados
  • Cifrado en tránsito: TLS 1.3 para todas las comunicaciones
  • Aislamiento de datos: Row Level Security (RLS) en PostgreSQL
  • Autenticación: Tokens JWT con expiración automática
  • Auditoría: Logs de acceso y modificaciones

ARTÍCULO 4. POLÍTICA DE RETENCIÓN

El Encargado se compromete a las siguientes prácticas de retención:

  • Documentos del usuario: Retenidos mientras el usuario mantenga su cuenta activa. Eliminación inmediata bajo solicitud.
  • Metadatos de análisis: Eliminación automática según la configuración del usuario (7, 15 o 30 días de inactividad).
  • Logs de facturación: Retenidos durante el período legal obligatorio (5 años según normativa fiscal española).

El usuario puede solicitar la eliminación total de sus datos en cualquier momento desde el panel de configuración de su cuenta.

ARTÍCULO 5. NO ENTRENAMIENTO DE IA

El Encargado garantiza expresamente que:

  • Los documentos del usuario NO se utilizan para entrenar modelos de inteligencia artificial propios ni de terceros.
  • Cada análisis es procesado de forma aislada y efímera.
  • OpenAI (subencargado) opera bajo su propio DPA que prohíbe el uso de datos de API para entrenamiento.

ARTÍCULO 6. DERECHOS DEL INTERESADO

El Encargado asistirá al Responsable para garantizar el ejercicio de los siguientes derechos:

  • Acceso: El usuario puede descargar todos sus datos desde la plataforma.
  • Rectificación: El usuario puede modificar su información de cuenta.
  • Supresión: Eliminación inmediata de datos bajo solicitud.
  • Portabilidad: Exportación de análisis en formato Excel/JSON.
  • Oposición: El usuario puede eliminar su cuenta completamente.

ARTÍCULO 7. NOTIFICACIÓN DE BRECHAS

En caso de violación de seguridad que afecte a datos personales, el Encargado notificará al Responsable en un plazo máximo de 48 horas desde que tenga conocimiento del incidente, proporcionando:

  • Descripción de la naturaleza de la violación
  • Categorías y número aproximado de interesados afectados
  • Medidas adoptadas para mitigar los efectos
  • Datos de contacto del Delegado de Protección de Datos (si aplica)

ARTÍCULO 8. SUBENCARGADOS

El Encargado utiliza los siguientes subencargados autorizados:

  • Supabase Inc. - Infraestructura de base de datos y autenticación
  • OpenAI LLC - Procesamiento de lenguaje natural
  • Resend Inc. - Servicio de correo electrónico transaccional
  • Stripe Inc. - Procesamiento de pagos

Todos los subencargados operan bajo acuerdos de procesamiento de datos compatibles con el RGPD.

ARTÍCULO 9. DURACIÓN Y TERMINACIÓN

Este Acuerdo entra en vigor en el momento de la creación de la cuenta y permanece vigente mientras el Responsable mantenga una relación de servicio con Helios.

Tras la terminación del servicio, el Encargado eliminará todos los datos personales en un plazo de 30 días, salvo obligación legal de conservación.

ARTÍCULO 10. JURISDICCIÓN

Este Acuerdo se rige por la legislación española y el Reglamento General de Protección de Datos (UE) 2016/679. Cualquier disputa será sometida a los tribunales de Castellón, España.
